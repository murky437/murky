Minimal server deployment.

 Key points:
 - no source code
 - only required binaries
 - logs
    - managed and rotated by caddy
 - keep n previous deployments
    - adding new prunes older ones
 - deploy.json file for info about deployment
    - timestamp
    - git commit hash
 - database backups separate
    - script prunes old ones
